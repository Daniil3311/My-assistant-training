1. Какое из этих НЕ является преимуществом Django?

Правильный ответ: Медленная обработка запросов. Django оптимизирован для быстрой разработки и имеет хорошую производительность.

2. Когда QuerySet выполняет запрос к базе данных?

Правильный ответ: При итерации по результатам. QuerySet "ленивый" - запрос выполняется только когда реально нужны данные.

3. Что из этого можно сделать с помощью Django ORM?

Правильный ответ: Все перечисленное. Django ORM поддерживает создание индексов через Meta.indexes и может выполнять сложные операции.

4. В чем разница между annotate() и aggregate()?

Правильный ответ: annotate() добавляет вычисляемые поля к каждому объекту, aggregate() возвращает словарь с общими значениями.

5. Что такое F-выражения в Django ORM?

Правильный ответ: F-выражения позволяют ссылаться на значения полей модели непосредственно в запросах.

6. Какой шаблон проектирования использует Django?

Правильный ответ: Django использует MTV (Model-Template-View), что похоже на MVC, но с другими названиями компонентов.

7. Для чего нужен метод dispatch() во View?

Правильный ответ: Метод dispatch() определяет, какой метод (get, post и т.д.) должен обработать запрос.

8. Что такое Django Channels?

Правильный ответ: Django Channels добавляет поддержку WebSockets, long-polling и других протоколов в реальном времени.

9. Что означает "батарейки в комплекте" в Django?

Правильный ответ: Django включает множество готовых компонентов (аутентификация, админка, ORM и т.д.), что ускоряет разработку.

10. Какой HTTP-метод используется для создания ресурса в REST API?

Правильный ответ: POST используется для создания ресурса, PUT - для полного обновления существующего.
